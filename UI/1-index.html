The barrier

Disconnected data silos

South Africa has a rich ecosystem of public pharmaceutical data, but it exists in independent silos. Analysts waste time manually extracting, cleaning, and rebuilding datasets. Our engine bridges these specific sources to create a unified view.

SAHPRA Registry

Regulatory data on active licensed pharmaceutical establishments.

  • Drug manufacturers
  • Registered packers
  • License validity

NDoH Procurement

Public sector procurement data from the Master Procurement Catalogue.

  • Master Health Product List
  • Unit prices & quantities
  • Supplier awards

Medicine Price Registry

Pricing data reflecting the private sector environment.

  • Private sector pricing
  • Molecule competitiveness

Under the hood

The MCP architecture

To move beyond static spreadsheets, we built a Model Context Protocol (MCP) server. It acts as a standardized gateway allowing large language models to interact with our local datasets dynamically, directly inside tools like Claude.

Live Data Ingestion

The engine pulls directly from three national sources on demand: the SAHPRA establishment registry, the NDoH Master Health Product List, and the Medicine Price Registry. Each source is fetched via a live HTTP request so the data reflects the most current published release, not a static snapshot.

Context-aware data architecture

The engine uses a multi-stage approach to work within the finite context windows of current LLMs. Rather than dumping raw datasets into the model, it implements "data valves" — configurable pagination limits (default: 15 rows per tool call) that keep individual responses focused and precise. For broad market questions, the engine's aggregation layer pre-summarises datasets by supplier, molecule, or care level before passing results to the model, dramatically reducing token overhead without sacrificing analytical depth.

The practical effect: the engine analyses the market in deliberate, verifiable slices, trading the illusion of one-shot completeness for genuine data integrity across every query.

Where this is going

Architectural roadmap: overcoming throttling

Moving to a hybrid retrieval architecture, combining semantic vector search with structured aggregation, will allow the engine to answer exploratory market questions across the full national dataset while preserving the exact-match precision required for regulatory and compliance queries.

Current state: throttled

  • Limited by LLM context windows.
  • Provides a high-fidelity "snapshot" of the market at a single point in time.
  • Analyses data in isolated "slices."
THE GOAL

Future state: hybrid retrieval

  • Selective retrieval: extracts only the most relevant fragments from indexed datasets.
  • Longitudinal analysis: efficiently compares multiple years of historical data simultaneously.
  • Scalable platform: capable of analysing trends across the entire national pharmaceutical landscape.
